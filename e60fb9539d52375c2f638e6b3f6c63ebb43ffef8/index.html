과정설정에는 총 강의시간인 8시간중 6시간은 강의 2시간은 인터뷰 및 금일 공부한 용정리시간을 주셨으며 각자 정리 및 실제 배운것을 실습하는 시간입니다.

또한 4~5명이 한팀이 되어 팀 프로젝트를 만들어야하는데 중요한점은 팀 프로젝트 주제 정하기와 분업을 어떻게 할것인지를 결정하는것 같습니다.

교육과정은 비전공자를 대상으로 하는것이라 난이도가 매우 낮았습니다, 말 그대로 기초과정이 많았네요, 강사님께서도 미리 예습해오는것을 권장하셨으며 예습을 해도 일단 강사님 수업은 예정대로 진행하신다고 하였습니다.

2019.12.24

오늘은 HTML을 보면서 가장 궁금했었던 div에 대해서 배웠습니다, 또한 github를 활용하여 개인 블로그를 작성했으며 특이사항으로는 기존에 미리 만들어주었던 html을 활용하여 git hub로 호스팅을 했다는 점입니다.

과정은 다음과 같이 진행했습니다.

  • 다른 사람이 만들어놓은 HTML 팜플렛을 수정하여 모양을 바꾼다.
  • Github에 새로운 프로젝트를 시작한다 (website.io)
  • Github의 Settings에 들어가 내리다보면 있는 Choose Theme로 아무거나 고른다.
  • https://아이디.github.io/프로젝트이름/ 내가 만든 사이트는 https://wjsxkxk1.github.io/website/이다.
  • 위 사이트에 접속되는지 확인하자 보통 2~3분정도 걸릴수도 있다.
  • 접속이 되었으면 GitHub Desktop을 설치한다.
  • 기존에 만든 프로젝트(website)를 File-clone repository으로 불러온다.
  • 불러온 프로젝트에서 우클릭을 하면 나오는 show explore으로 폴더를 열어준다.
  • 해당 폴더에 내가만든 .html과 css나 js파일이 있다면 같이 넣어준다.(폴더째로 넣어도 된다)
  • GithubDesktop 우상단의 Push Origin으로 파일을 최신화 해준다.파일=>Desktop Github=>commit to master=>GitHub
  • 좌하단의 Commit to master을 누른다.
  • 4번째 항목의 사이트 주소로 들어가 사이트가 최신화 됬는지 확인한다.

Day 목록으로

12월 25일은 크리스마스 공휴일이라 수업을 쉬었고 26일이 3일차 수업이 되었다. 금일 수업은 기본적인 도메인 구축과 미리 만들어 놓은 html를 index.html로 업로드해서 실제 구동이 가능한 웹서버를 구축하고 이를 관리하기 위해 filezilla FTP관리프로그램을 사용하게 되었다, 물론 직접 파일을 해당 웹 폴더에 넣어도 동작은 하긴한다.

예전에 만들어 놓았던 html 파일을 업로드하였고 추가로 GitHub에 해당 html을 업로해서 웹페이지를 만들었다, 실제로 해보니 Github나 dothome에서 주어진 서버나 업로드하는 방식이 다를뿐이지 동작은 동일하였다.

목적


해당 페이지에 접속하는것 까지는 좋은데 비밀로하고 싶은 페이지에 패스워드를 걸고 싶었다, 물론 코드파일자체에다 form을 활용해서 암호를 거는 방법도 있지만 개발자도구로 코드를 확인하면 바로 노출되는 문제가 있어 여러 방법을 찾아보았다, htaccess방법도 있지만 동작이 안되 포기하고 다른 방법들을 찾고 있다.

4일차

Day 목록으로

전반적으로 수업이 "루즈"해진 느낌이었다, 거의다 아는것이었고 자세하게 설명을 안해주셔도 구글링하면 다 알수 있는것들이라 넘어가 주셨으면 했는데 몇몇 인원이 수업속도를 못따라와 많이 느려진 느낌이다.

자바스크립트를 활용하는 방법들에 대해 배웠고 전체적으로 <\script>를 많이 사용하였다, 하지만 왠만한 프로그램들은 js를 따로 외부파일로 빼놓고 사용하는경우가 많으니 해당 방법을 운용하는것을 아는게 더 중요한것 같다.

성공한것은 웹페이지에 별도의 패스워드를 부여하는 방법을 찾았다. 방법은 아래와 같다.

  • 패스워드가 적용하기위한 최초페이지 index.html을 https://github.com/matteobrusa/Password-protection-for-static-pages 에서 받아온다.
  • index를 실행한후 원하는 암호를 입력한다(단 숫자는 입력되지 않는다)

  • wrong password가 뜬것을 확인한후 주소창에서 #이후의 무작위 값들을 복사한다.(#은 제외한다) ex) #f21rk5o2k2o1pk41po24 [<-해쉬값이다.]

  • 해당 해쉬값을 이름으로하는 폴더를 생성한다[단 폴더의 위치는 index.html과 동일하다/.

  • 만든 폴더에 웹페이지 html과 asset들을 전부 넣어준다.[단, 파일명은 index.html이여야한다.]

  • 깃허브에 업로드한다

  • 최초 홈페이지를 만들었던것과 같이 choose theme를 통해 주소값을 받는다.

  • 받은 주소값을 실행하면 패스워드가 적용된 홈페이지가 열린다.

오늘은 간단하게 자바스크립트로 별삼각형들을 만들고 경고창으로 뜨게하는것을 숙제로 받았다. 난이도는 매우 쉬움정도이지만 처음 코딩할경우에는 굉장히 어려울것이다. 파이썬이나 c라면 간단하게 1개씩 표시하면 되겠지만 이놈은 한번에 모두 표시되야해서 1개씩 저장해야되고 저장되는것도 각각의 반복문 사이에 추가해야해서 잘 생각해야했다, 쉽게 말해 더럽게 귀찮았다.
